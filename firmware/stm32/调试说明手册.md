# 《2026-06-13》系统状态与功能总结

### 当前工程配置状态

- **系统类型**: 六舵机PWM控制系统 + 温湿度监控 + 风速风向监控 + 继电器控制 + 3.5寸 TFTLCD 触摸显示
- **开发环境**: Keil uVision + STM32F103ZE
- **通信接口**: USART1 (PA9/PA10) @ 115200bps
- **显示模块**: 普中 STM32 TFT 3.5寸彩屏（HX8357DN，320×480 竖屏）+ XPT2046 电阻触摸（FSMC 并口 + 软件 SPI 触摸）
- **硬件模块**: SI7021温湿度传感器 + SM5388风速风向传感器 + 六路舵机 + 继电器 + TFTLCD
- **最后更新**: 2026年6月13日
- **本次升级**: 由 0.96" OLED（PB12/PB13 软件 I2C）切换到 3.5寸 TFTLCD（FSMC 并口 + XPT2046 触摸）。OLED 源文件保留但工程已排除编译；触摸校准数据通过内部 Flash 最后一页（0x0807F800）模拟 EEPROM 持久化，无需外置 AT24CXX 芯片。当前 UI 已启用触摸输入与动态刷新。
- **移植历史**: 由 STM32F103C8 (中密度/64KB Flash/20KB RAM) 移植至 STM32F103ZE (大密度/512KB Flash/64KB RAM)

### 核心功能特性

1. **六舵机独立控制**: 支持PA1/PA2/PA3/PB6四路 + PB7/PB9两路 PWM输出
2. **舵机1自动旋转**: 可在手动/自动模式间切换，自动模式10秒循环0-180-0度
3. **box控制**: 串口收到box=1时激活servo5转到100度、servo6转到25度；box=0时servo5转回25度、servo6转回100度；边沿触发，不重复动作
4. **继电器控制**: PB8引脚控制，支持高电平启动
5. **温湿度监控**: SI7021传感器，软件I2C接口(PA4/PA5)
6. **风速风向监控**: SM5388传感器，ADC模拟量输入(PA6/PA7)
7. **3.5寸 TFTLCD 触摸显示**: HX8357DN 320×480 彩屏（FSMC 并口）+ XPT2046 电阻触摸（软件 SPI），开机动画 + 实时数据看板
8. **双协议支持**: JSON命令 + 传统字符串命令
9. **实时状态回报**: 四类数据错时回报（角度/温湿度/风速风向/模式+box）
10. **触摸校准持久化**: 校准数据写入内部 Flash 最后一页（0x0807F800），断电不丢失

### 舵机上电默认角度

- **舵机2**: 上电默认0度，传统 S2HOME/ALLHOME 归零到0度；JSON `{"action":"home"}` 会归到90度，工作范围0-180度
- **舵机3**: 上电默认90度，home归零到90度，**限位90-180度**
- **舵机4**: 上电默认0度，home归零到0度，工作范围0-180度
- **舵机5**: 上电默认25度，box=1时转到100度，box=0时回到25度（PB7）
- **舵机6**: 上电默认100度，box=1时转到25度，box=0时回到100度（PB9）

> 画 PCB 时建议：
> - 电源：VCC5（Pin 1）和 VCC3.3（Pin 33）二选一即可，模块板载有稳压；如果你的主板已经有稳定 3.3V，优先接 Pin 33 省一级稳压。两端 GND（Pin 2、Pin 34）都焊上，回流路径更短。
> - 数据线：D0-D15 共 16 根，建议平行走线、等长（同层不超过 ±5mm 差），离 NWE/NOE 至少 8mil 间距。
> - 控制线：CS/RS/WR/RD 走在数据线同侧，T_CS/T_SCK/T_MOSI/T_MISO 单独走一组。
> - RST：直接接 MCU 的 NRST 网络即可，不需要单独 GPIO。
> - BL 背光：从 PB0 串个 100Ω 限流电阻再到 Pin 3，防止上电瞬态。

### 主循环调度与回报机制

- 主循环每50ms自增tick，tick周期为20（即1秒循环一次）
- 四类JSON数据在1秒内各自回报一次，且错开时序：
  - tick == 0：回报四舵机角度（servo1/2/3/4）
  - tick == 5：回报温湿度（temp/humi）
  - tick == 10：回报风速风向（su/xiang）
  - tick == 15：回报模式状态（auto/servo1stop/switch），紧接回报box状态

### JSON回报格式举例

```json
{"servo1":90.0,"servo2":0.0,"servo3":90.0,"servo4":0.0}
{"temp":25.36,"humi":60.12}
{"su":12.50,"xiang":"N"}
{"auto":1,"servo1stop":0,"switch":0}
{"box":0}
```

### 外设模块

#### 串口通信配置

- **接口**: USART1 (PA9-TX, PA10-RX)
- **波特率**: 115200 bps
- **数据位**: 8位，停止位：1位，校验位：无，流控：无
- 建议使用USB转TTL模块进行调试，注意GND共地

#### 3.5寸 TFTLCD 触摸屏

- **屏幕**: 普中 STM32 TFT 3.5寸彩屏，HX8357DN 控制器，320×480 竖屏
- **接口**: FSMC 16-bit 并口，挂在 Bank1 NORSRAM4，命令地址 0x6C000000、数据地址 0x6C000800（A10=1）
- **触摸**: XPT2046 电阻屏，软件 SPI 驱动
- **背光**: PB0 GPIO 推挽控制，高电平点亮
- **校准**: 首次上电会进入 5 点十字校准，校准数据写入内部 Flash 最后一页（0x0807F800），断电不丢失
- **显示内容**（竖屏 320×480 看板，触摸导航与动态刷新已启用）:
  - y=0~39   蓝色标题栏 "STM32 Control"
  - y=50~130 舵机区：S1 / S2 / S3 角度 + 继电器 ON/OFF
  - y=140~220 传感器区：T（温度）/ H（湿度）
  - y=230~310 风速风向区：WS（风速）/ WD（风向）
- **接线**: 详见上方"3.5寸 TFTLCD 屏幕"和"XPT2046 电阻触摸"两张表，普中模块插板对位即可

> ⚠️ 原版 0.96" OLED（PB12/PB13 软件 I2C）已停用，但源文件 OLED.c / OLED_Animation.c / OLED_Font.h 仍保留在 Hardware 目录，工程文件已通过 `<IncludeInBuild>0</IncludeInBuild>` 排除编译。需要回滚到 OLED 时，恢复编译 + 替换 main.c 初始化即可。

#### SM5388风速风向传感器

- **测量范围**: 风速0-30m/s，风向0-360°
- **输出方式**: 0-5V模拟电压，**需分压到3.3V后接ADC**
- **ADC接口**: PA6(风速)、PA7(风向)
- **数据上报**: `{"su":风速,"xiang":"方位"}`，方位为N/NE/E/SE/S/SW/W/NW

**⚠️ 分压电路（每通道独立）**:

```
SM5388输出(0-5V) ----[10kΩ]----+---- STM32 PA6/PA7
                                |
                            [20kΩ]
                                |
                               GND
```

分压比：Vout = Vin × 0.667，5V最大输出3.33V，安全。

#### SI7021温湿度传感器

- **I2C接口**: SDA→PA5，SCL→PA4（软件模拟I2C）
- **供电**: 3.3V推荐，GND共地
- SDA/SCL建议各加4.7kΩ上拉电阻到3.3V

#### 继电器

- **控制引脚**: PB8
- **触发方式**: 高电平启动，低电平关闭
- 需独立供电，GND与STM32共地

## ❗ 常见问题排查

### 舵机不响应

1. 检查角度值是否在有效范围内（servo3限位90-180度）
2. 确认PWM引脚连接正确（PA1/PA2/PA3/PB6/PB7/PB9）
3. 检查舵机独立供电是否正常（舵机不可直接由STM32 3.3V供电）
4. 确认舵机1当前是否处于自动模式（自动模式不响应角度命令）

### JSON解析失败

1. 检查JSON格式是否正确（括号、引号、逗号）
2. 确认末尾有 `\r` 或 `\n`
3. 发送数据是否完整（串口助手缓冲区问题）

### 温湿度读取异常

1. 检查PA4/PA5接线
2. SDA/SCL需加4.7kΩ上拉电阻到3.3V
3. 确认SI7021供电为3.3V，GND共地
4. 串口输出 `{"error":"SI7021 sensor read error"}` 则为I2C通信失败

### 风速风向读数异常

1. 检查PA6/PA7分压电路，确保输出不超过3.3V
2. 用万用表测量分压后电压，正常范围0-3.3V
3. GND必须与STM32共地
4. ADC读数为0或4095则检查传感器供电和接线

### 串口无输出

1. 确认PA9(TX)→USB转TTL的RXD，PA10(RX)→TXD，GND共地
2. 波特率确认115200
3. 检查是否有开机JSON状态信息输出

### TFTLCD 无显示 / 花屏 / 显示错乱

1. **完全黑屏 / 不亮**：先确认 PB0（背光 BL）已经被拉高；TFTLCD_Init() 里会自动初始化背光为高电平，万用表测 PB0 应≈3.3V。
2. **背光亮但屏幕白屏 / 雪花**：FSMC 数据/控制线虚焊。重点检查 PD0/PD1/PD4/PD5/PD8/PD9/PD10/PD14/PD15、PE7~PE15、PG0(RS)、PG12(CS)，任意一根虚焊都会导致花屏。
3. **显示偏色 / 颜色反相**：通常是 LCD ID 没识别上、走到了错误的初始化序列。串口会打印 `LCD ID:xxxx`，正常应为 `8357`（HX8357DN）；不对就检查 RD（PD4）/WR（PD5）。
4. **位置错乱 / 镜像**：tftlcd.c 默认竖屏（320×480）。如果方向不对可调用 `LCD_Direction(0/1/2/3)` 切换。
5. **程序卡死**：模块未上电或 RST 被拉低。普中模块 RST 默认接 STM32 复位脚，烧录后会自动复位。

### 触摸不响应 / 校准异常

1. **完全不响应**：检查 PB1(TCLK) / PB2(DOUT) / PF9(TDIN) / PF10(PEN) / PF11(TCS) 接线。PEN 是触笔检测信号，按压时应被拉低；用万用表按压时测量 PF10，正常会从 3.3V 降到 0V。
2. **首次开机进入校准**：HX8357DN 默认走电阻屏 XPT2046 路径。第一次上电校准数据全 0xFF，会自动进入 5 点十字校准界面，依次按完中心+四角即可。
3. **重启后又进校准**：内部 Flash 写入失败。检查 24cxx.c 的 `eep_flush_to_flash()` 是否被调用；最后一页地址 0x0807F800 必须和 STM32F103ZE（HD，2KB 页大小）匹配。
4. **触摸位置偏移**：长按屏幕背后的复位/校准触发，或在 touch.c 里临时把 magic 改成非 0x0A 强制重新校准。
5. **触摸抖动 / 误报**：tp_dev.scan() 内部已做去抖；如果环境噪声大，可在外壳上加屏蔽层，或把 PB1/PB2 走线远离 PWM。

```c
    Serial_Printf("{\"new_key_set\":%d}\r\n", (int)new_item->valuedouble);
}
```
